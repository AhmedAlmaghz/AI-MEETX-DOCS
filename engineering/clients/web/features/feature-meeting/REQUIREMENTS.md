# Web Meeting UI Requirements

Document ID: WEB-MEETING-REQ-001
Version: 1.0.0
Status: Approved

---

# 1. Purpose
Defines the functional requirements for meeting room dashboards, lobby screens, and hosts controls.

---

# 2. Functional Requirements

- **WM-FR-001 (Lobby)**: The client SHALL display a staging screen before meeting entry, allowing video preview toggles.
- **WM-FR-002 (Admission)**: If a waiting room is enabled, participants SHALL see a staging screen until host admits them.
- **WM-FR-003 (Host Controls)**: The host dashboard SHALL render action buttons (Mute All, Mute participant, Admit entry, Kick participant).
- **WM-FR-004 (Roster)**: The participant panel SHALL show attendees, hand raising orders, and mic statuses.
- **WM-FR-005 (Presence)**: The UI list SHALL mark users disconnected or away if heartbeats drop out.
