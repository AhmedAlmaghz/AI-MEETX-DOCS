# 1. Overview

The `feature-ai` module provides an intelligent in-meeting assistant powered by Google Gemini APIs.

It listens to the meeting transcript (generated by `feature-translation`), analyzes conversation context, and delivers:

- Real-time AI-generated meeting summaries
- Action item detection and extraction
- Smart Q&A assistance
- Meeting topic analysis
- Post-meeting automated reports

---

# 2. Business Requirements

## 2.1 Real-Time Intelligence

| ID | Requirement |
|----|-------------|
| AI-BR-001 | The system MUST generate a running meeting summary that updates in real-time as the conversation progresses. |
| AI-BR-002 | The system MUST detect and extract action items when participants assign tasks during the meeting. |
| AI-BR-003 | The system MUST allow participants to ask questions to the AI assistant using in-meeting chat. |
| AI-BR-004 | AI usage MUST be clearly disclosed to all participants by a visible in-meeting indicator. |

## 2.2 Post-Meeting Reports

| ID | Requirement |
|----|-------------|
| AI-BR-010 | After meeting ends, the system MUST generate a full post-meeting report including: summary, decisions made, action items (with owners), and topic breakdown. |
| AI-BR-011 | The post-meeting report MUST be available within 5 minutes of meeting end. |
| AI-BR-012 | The report MUST be exportable as Markdown and PDF. |

---

# 3. Functional Requirements

## 3.1 Transcript Processing

| ID | Requirement |
|----|-------------|
| AI-FR-001 | The AI module MUST consume transcript segments from `feature-translation`. |
| AI-FR-002 | Processing MUST occur in a streaming fashion — not requiring the full transcript before starting. |

## 3.2 AI Q&A Chat

| ID | Requirement |
|----|-------------|
| AI-FR-010 | Participants MUST be able to type questions prefixed with `@AI` in the meeting chat to invoke the assistant. |
| AI-FR-011 | The AI MUST respond within 3 seconds using context from the last 30 minutes of transcript. |

## 3.3 Privacy & Consent

| ID | Requirement |
|----|-------------|
| AI-FR-020 | Transcript data used for AI analysis MUST NOT be stored permanently unless explicitly enabled by the host. |
| AI-FR-021 | Post-meeting reports MUST only be accessible to the meeting host and co-host by default. |

---

# 4. Non-Functional Requirements

## 4.1 Performance

| ID | Metric | Target |
|----|--------|--------|
| AI-NFR-001 | AI summary update latency | < 5 seconds per update cycle |
| AI-NFR-002 | Q&A response latency | < 3 seconds p95 |
| AI-NFR-003 | Post-meeting report generation | < 5 minutes |

## 4.2 AI Governance

| ID | Requirement |
|----|-------------|
| AI-NFR-010 | ONLY Gemini APIs may be used. No third-party AI models are permitted. |
| AI-NFR-011 | All Gemini calls MUST go through a central AI Gateway with rate limiting. |

---

# 5. Dependencies

- `feature-translation` -> For transcript segments (consumed via events).
- `feature-meeting/lifecycle` -> To trigger post-meeting report generation.
- `feature-chat` -> To receive @AI chat queries and send AI responses.
